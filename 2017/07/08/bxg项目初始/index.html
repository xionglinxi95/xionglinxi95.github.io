session和cookie分析

session(服务器)是属于后台（服务器端的技术）的，cookie(客户端)是属于前端（浏览器的技术）的
  • http协议是无状态的（状态一种记录的行为，例如你登录一个网站）
  • 但是实际情况需要维护客户端与服务器之间的状态
  • 可以通过session和cookie来解决这个问题

cookie基本操作

  • 分类：
    • 会话cookie(session cookie)
      • 只会保留在浏览器中，不会保留在文件中，当浏览器关闭则不再
    • 持久cookie(persistan cookie)(持久的cookie会把cookie内容存储到浏览器指定的文件中)–》添加expires

  • var data = new Date();
    data.setDate(data.getDate()+7);
    // 当前日期的后面7天
    document.cookie=’hello=world; expires=’+data.toGMTString();
    // 这个时间是cookie的特定的时间格式

    document.cookie=’hello=world; expires=’+data.toGMTString()+ ‘; path=/‘;注意一下这里的格式
    //设置的根路径，所以在所有根路径下的文件都能访问到，当然path的路径也是可以修改的，只有在path设置的那个路径
    // 的下面才可以访问到之前所设置的cookie,否则访问不到，如果不设置path,则默认path的值是当前文件的上一级目录，只有在上一级目录下的内容才可以被访问到这个cookie的内容，在这个目录上或者同级都将访问不到。
    // console.log(document.cookie);

  • 获取cookie—-> document.cookie

  • 设置cookie—->document.cookie=’username=list’
  • cookie的设置是累加的，之前设置的内容不会丢失（但是在关闭浏览器的时候内容内容消失，输出的情况下只是一个空字符串），cookie之间是用； 来分割的（逗号与空格）
  • cookie的访问规则如下(在浏览器的设置里面可以看到访问页面的cookie以及cookie的日期和路径)

  • 获取指定的cokie的值（获取的cookie值都是字符串）—–》利用“; ”来分割

  • 利用cookie.js插件的api的使用，removeCOOkie()的时候如果不设置path则删除当前的路径下的指定的名字。设置的话则根据设置的来说删除。

    框架：一些通用功能的（例如一个框架可以使用在这个网站也可以使用在哪个王炸）
    库：提供方法
    ps:但是框架和库在前端方面没有什么特别的区别

    组件，插件，控件都是基于框框架和库的基础上的
